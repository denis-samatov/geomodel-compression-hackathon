# Базовое решение

Этот каталог содержит минимально валидное эталонное решение для хакатона.

Базовое решение нужно не для победы в рейтинге, а для того, чтобы участник мог:

- понять обязательные требования к решению;
- увидеть рабочий формат `compress.py` и `decompress.py`;
- проверить локальную совместимость с публичным оценщиком;
- быстро запустить сквозной сценарий быстрой проверки перед собственной реализацией.

## Профиль базового решения

| Блок | Описание |
|---|---|
| Метод | `tar-gz-full-copy` |
| Цель | Показать минимально валидную реализацию без доменной оптимизации |
| Зависимости | Только стандартная библиотека Python |
| Стратегия упаковки | Полная упаковка дерева модели в `tar.gz` |
| Стратегия восстановления | Полная распаковка архива обратно в каталог |
| Назначение | Эталонное решение и быстрая проверка инфраструктуры |

## Что гарантирует решение

| Гарантия | Значение |
|---|---|
| Соответствие требованиям | Соблюдает обязательный интерфейс командной строки |
| Сохранение всего дерева | Копирует все файлы модели без выборочного отбора |
| Отсутствие внешней сети | Не требует сетевого доступа |
| Воспроизводимое поведение | Работает детерминированно в чистом окружении |
| Совместимость с публичным оценщиком | Подходит для локальной проверки через `public_evaluator` |

## Что это базовое решение не пытается делать

| Ограничение | Почему это важно |
|---|---|
| Не делает предметно-ориентированного сжатия | Это не соревновательное решение, а минимальный эталон |
| Почти не уменьшает размер модели | Полный перенос дерева нужен для демонстрации минимальных требований |
| Не оптимизирует время выполнения | Цель базового решения не в производительности, а в корректности |
| Не является верхней границей по качеству | Это нижняя граница валидности, а не ориентир для победы |

## Перечень файлов (согласно требованиям)

| Файл | Назначение |
|---|---|
| `compress.py` | Упаковывает модель целиком в `tar.gz` |
| `decompress.py` | Распаковывает артефакт обратно в каталог |
| `config.yaml` | Краткая конфигурация эталонного решения |
| `submission.json` | Метаданные базового решения в формате участника |
| `requirements.txt` | Фиксирует, что базовое решение использует только стандартную библиотеку |
| `tests/README.md` | Объясняет подход к быстрой проверке базового решения |

## Обязательный интерфейс

```bash
python compress.py --input <path_to_model> --output <path_to_archive>
python decompress.py --input <path_to_archive> --output <path_to_restored_model>
```

## Быстрый старт

```bash
python baseline/compress.py --input /path/to/model --output /tmp/artifact.tar.gz
python baseline/decompress.py --input /tmp/artifact.tar.gz --output /tmp/restored-model
```

## Быстрая проверка через публичный оценщик

```bash
python public_evaluator/evaluate.py \
  --model /path/to/public-model \
  --solution-dir baseline \
  --workdir /tmp/baseline-eval
```

## Ожидаемые артефакты

| Этап | Что формируется |
|---|---|
| После `compress` | `artifact.tar.gz` и `artifact.tar.gz.metadata.json` |
| После `decompress` | Каталог восстановленной модели и `baseline_restore_metadata.json` |

## Как это работает

1. `compress.py` рекурсивно обходит каталог модели.
2. Все найденные файлы упаковываются в единый `tar.gz`.
3. Рядом с архивом записывается файл метаданных с базовой статистикой.
4. `decompress.py` распаковывает архив в целевой каталог.
5. После восстановления записывается файл метаданных о результате распаковки.

## Как участникам следует использовать базовое решение

- не копируйте решение как финальное без изменений;
- используйте его как минимальный стартовый шаблон;
- проверяйте через него, что ваш локальный конвейер и оценщик работают корректно;
- сравнивайте собственную реализацию с базовым решением по корректности, а не по качеству сжатия.
