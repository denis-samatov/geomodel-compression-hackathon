# Публичный оценщик

Этот раздел содержит локальный публичный оценщик для участников.

Публичный оценщик должен давать достаточно информации для разработки, но не раскрывать приватную валидацию.

## Что он делает

| Возможность | Назначение |
|---|---|
| Проверки интерфейса командной строки | Быстро выявляет нарушения обязательного интерфейса |
| Быстрая валидация | Отсекает очевидно некорректные решения до отправки решения |
| Публичная оценка | Дает участнику ориентир для итераций |
| Ограниченная детализация | Помогает улучшать решение без раскрытия скрытой логики |

## Доступные файлы

- `evaluate.py` — основной локальный оценщик;
- `metrics.py` — расчет базовых публичных метрик;
- `contract_checks.py` — минимальные проверки обязательного контракта;
- `tnavigator_check.py` — дополнительная локальная проверка открытия восстановленной модели в tNavigator;
- `sample_inputs/` — место для малого датасета быстрой проверки.

## Быстрый старт

Запуск на готовом решении:

```bash
python public_evaluator/evaluate.py \
  --model /path/to/public-model \
  --solution-dir /path/to/team-solution \
  --workdir /tmp/evaluator-run
```

Запуск с дополнительной проверкой открытия восстановленной модели в tNavigator:

```bash
python public_evaluator/evaluate.py \
  --model /path/to/public-model \
  --solution-dir /path/to/team-solution \
  --workdir /tmp/evaluator-run \
  --tnavigator-exe "C:\Program Files\RFD\tNavigator\24.2\tNavigator-con.exe" \
  --tnavigator-mode smoke \
  --tnavigator-ignore-lock
```

Запуск на уже собранном артефакте:

```bash
python public_evaluator/evaluate.py \
  --model /path/to/public-model \
  --archive /path/to/artifact.tar.gz \
  --restored-model /path/to/restored-model
```

## Что означает итоговая оценка

Оценщик считает упрощенный `public_score` на основе:

- коэффициента сжатия;
- полноты восстановления структуры;
- доли точных совпадений для совпадающих файлов;
- суммарного времени выполнения.

Эта оценка нужна как локальный ориентир и не заменяет финальную приватную валидацию организаторов.

## Дополнительная проверка через tNavigator

Если указан `--tnavigator-exe`, публичный оценщик после восстановления модели дополнительно пытается открыть восстановленный проект через tNavigator.

Эта проверка полезна для локальной валидации, потому что:

- подтверждает, что восстановленный проект действительно читается в tNavigator;
- помогает поймать ошибки в `.DATA`, `INCLUDE/`, `USER/` и связях между файлами;
- читает `result.log`/`results.log` из `RESULTS` и извлекает из него итоговый статус, время выполнения и сводку ошибок;
- дает отдельный структурированный отчет со статусом запуска, предупреждениями и путями к логам.

Важно:

- проверка является опциональной и не запускается без явного `--tnavigator-exe`;
- она рассчитана на локальную среду с установленным tNavigator и не должна считаться обязательной частью кроссплатформенной проверки;
- если проверка включена и tNavigator не может открыть восстановленную модель или в `result.log` зафиксированы ошибки/проблемы, итоговый код завершения `evaluate.py` будет ненулевым.
