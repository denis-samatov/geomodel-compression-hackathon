# Контракт подачи решения 

## Назначение документа

Этот документ задает точный технический контракт подачи решения: обязательную структуру репозитория, интерфейс командной строки и минимальные предварительные проверки.

Команды-участники хакатона, выбравшие задачу эффективного сжатия модели, должны поделиться с организаторами доступом к личному github-репозиторию с решением.

## Обязательная структура репозитория

```text
team-solution/
├── README.md
├── requirements.txt
├── compress.py
├── decompress.py
├── submission.json
├── config.yaml
└── tests/
```

Эталонная структура опубликована в [submission_template](../submission_template/README.md).

## Обязательные команды

```bash
python compress.py --input <path_to_model> --output <path_to_archive>
python decompress.py --input <path_to_archive> --output <path_to_restored_model>
```

## Ожидаемый результат

| Этап | Ожидаемые файлы |
|---|---|
| После `compress` | Архив или бинарный артефакт, `metadata.json`, журналы выполнения |
| После `decompress` | Каталог восстановленной модели и журналы выполнения |

## Что должно быть в файле README.md

- краткое описание подхода;
- инструкция по установке;
- инструкция по запуску;
- список зависимостей;
- перечень дополнительных артефактов, если они нужны.

## Частота подачи решений

| Правило | Что это значит |
|---|---|
| Дневной лимит | Рекомендуемый лимит: 2 отправки решения в день на команду |
| Политика таблицы результатов | Учитывается лучшая корректная подача решения команды за день |
| Невалидные попытки | Могут учитываться как попытки, если не объявлено иное |

## Формат поставки решения

Каждая подача решения должна быть оформлена как:

- репозиторий GitHub команды;
- архивная версия с тегом или архив решения;
- заполненный `submission.json`.

## Локальная среда тестирования

До отправки вы можете самостоятельно протестировать решение в среде, идентичной среде проверки:

```bash
cd starter-kit
docker build -t geomodel-judge .

docker run --rm \
    -v $(pwd)/../:/workspace \
    --network none \
    geomodel-judge \
    /bin/bash -c "cd /workspace && python public_evaluator/evaluate.py --model /workspace/<path_to_public_model> --solution-dir /workspace/<path_to_team_solution>"
```

## Проверки перед оценкой

- корректность структуры;
- наличие обязательных файлов;
- совместимость интерфейса командной строки;
- возможность поднять окружение;
- отсутствие ручных шагов.

## Условия немедленного отклонения решения

- отсутствует `compress.py` или `decompress.py`;
- нарушен контракт интерфейса командной строки;
- решение требует подключение к интернету;
- решение не воспроизводится в чистой среде;
- отсутствуют критичные зависимости или конфигурации.

Подробности про правила участия приведены в [rules.md](./rules.md), а информация про метрики и логику таблицы результатов — в [evaluation.md](./evaluation.md).
