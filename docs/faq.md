# Часто задаваемые вопросы

<details>
  <summary><strong>Что такое контракт?</strong></summary>
  <p>Говоря простыми словами, контракт — это чуть больше, чем образец структуры файлов решения. Это соответствие установленному шаблону и выполнение ряда проверок решения, которые опубликованы организаторами. Узнать подробнее о контракте можно здесь: <a href="./submission.md">submission.md</a>. </p>
</details>

<details>
  <summary><strong>Где находится официальная версия задания?</strong></summary>
  <p>В этом репозитории. Для краткого обзора используйте <code>README.md</code>, а для подробностей переходите в <code>docs/</code>.</p>
</details>

<details>
  <summary><strong>Можно ли использовать языковые модели?</strong></summary>
  <p>Да, как инструмент разработки. Ограничения и условия описаны в <a href="./rules.md">rules.md</a>.</p>
</details>

<details>
  <summary><strong>Можно ли использовать библиотеки с открытым исходным кодом?</strong></summary>
  <p>Да, если они совместимы с офлайн-исполнением и явно указаны в зависимостях. См. <a href="./rules.md">rules.md</a>.</p>
</details>

<details>
  <summary><strong>Можно ли использовать предобученные модели?</strong></summary>
  <p>Можно, если они воспроизводимы в офлайн-среде проверки и поставляются корректно. См. <a href="./rules.md">rules.md</a>.</p>
</details>

<details>
  <summary><strong>Можно ли требовать ручной запуск каких-то шагов?</strong></summary>
  <p>Нет. Автоматичность является обязательной и описана в <a href="./rules.md">rules.md</a> и <a href="./submission.md">submission.md</a>.</p>
</details>

<details>
  <summary><strong>Можно ли делать нестандартный интерфейс командной строки?</strong></summary>
  <p>Нет. Точный интерфейс зафиксирован в <a href="./submission.md">submission.md</a>.</p>
</details>

<details>
  <summary><strong>Можно ли отправлять решение много раз подряд?</strong></summary>
  <p>Ориентируйтесь на ограничения из <a href="./submission.md">submission.md</a> и на порядок публикации результатов из <a href="./evaluation.md">evaluation.md</a>.</p>
</details>

<details>
  <summary><strong>Почему таблица результатов не обновляется в реальном времени?</strong></summary>
  <p>Политика публикации результатов описана в <a href="./evaluation.md">evaluation.md</a>.</p>
</details>

<details>
  <summary><strong>Что делать, если локально решение работает, а система проверки отклоняет отправку решения?</strong></summary>
  <p>Сначала проверьте <a href="./submission.md">submission.md</a>, затем <a href="./evaluation.md">evaluation.md</a> и локальный публичный оценщик.</p>
</details>

<details>
  <summary><strong>Что такое корректная подача решения?</strong></summary>
  <p>Это подача решения, которая соответствует <a href="./submission.md">контракту подачи</a> и проходит обязательные проверки из <a href="./evaluation.md">evaluation.md</a>.</p>
</details>

<details>
  <summary><strong>Определяет ли публичная таблица результатов победителя?</strong></summary>
  <p>Нет. Итоговый рейтинг определяется приватной проверкой; подробности приведены в <a href="./evaluation.md">evaluation.md</a>.</p>
</details>

<details>
  <summary><strong>Где смотреть обновления по правилам и срокам?</strong></summary>
  <p>Правила смотрите в <a href="./rules.md">rules.md</a>, сроки — в <a href="./timeline.md">timeline.md</a>.</p>
</details>
