# Документация для участников

Этот каталог содержит детальную документацию хакатона. Корневой [README.md](../README.md) остается обзорной страницей, а здесь находятся описания отдельных тем.

> Если между внешними анонсами и этим репозиторием есть расхождения, приоритет имеет содержимое этого репозитория.

## Распределение тем

| Вопрос | Основной документ | Что в нем зафиксировано |
|---|---|---|
| Постановка задачи | [challenge-spec.md](./challenge-spec.md) | Постановка задачи, целевой результат и среда проверки |
| Правила участия | [rules.md](./rules.md) | Формат участия, разрешения, запреты и основания для дисквалификации |
| Даты и этапы хакатона | [timeline.md](./timeline.md) | Официальный календарь и обновление сроков |
| Что подается на вход алгоритму | [data-format.md](./data-format.md) | Структура модели, роли каталогов и публикация данных |
| Как должен выглядеть репозиторий решения | [submission.md](./submission.md) | Точные требования к структуре, интерфейсу и предварительным проверкам |
| Как сравниваются результаты | [evaluation.md](./evaluation.md) | Валидация, метрики и политика таблицы результатов |
| Где искать быстрые ответы | [faq.md](./faq.md) | Короткие ответы со ссылками на профильные документы |

## Рекомендуемый порядок чтения

1. Сначала прочитайте [спецификацию задания](./challenge-spec.md) и [правила](./rules.md).
2. Затем изучите [формат данных](./data-format.md) и [требования к решению](./submission.md).
3. После этого переходите к [логике оценки](./evaluation.md) и локальному оценщику.
4. Перед подачей решения проверьте [календарь](./timeline.md) и [FAQ](./faq.md).
